# Technical Specification

## Architecture
Single static HTML file (`index.html`) with inline `<style>` and `<script>` blocks. All external SDKs loaded via CDN.

## Data Flow
```
User Browser                    Firebase RTDB              Python TTS (local)
─────────────                   ─────────────               ─────────────────
1. Types text                                                          
2. Clicks Send                                                         
3. JS validates                                                       
4. push({text,ts,status}) ────> messages/<key>                        
                                  │                                    
5. Clear input                    │             6. on_value() listener 
                                  │                detects new child   
                                  │                                    
                                   <──────────── 7. Reads text         
                                                  8. TTS speaks it     
                                                  9. Updates status    
                                  <────────────    to "read"           
```

## CDN Dependencies

### Twitch Embed API
```html
<script src="https://embed.twitch.tv/embed/v1.js"></script>
```

### Firebase (modular, from gstatic CDN)
```html
<script type="module">
  import { initializeApp } from "https://www.gstatic.com/firebasejs/12.13.0/firebase-app.js";
  import { getDatabase, ref, push } from "https://www.gstatic.com/firebasejs/12.13.0/firebase-database.js";
</script>
```
Note: `firebase-analytics.js` is NOT needed. Only `firebase-app.js` and `firebase-database.js`.

## Twitch Player Events
| Event | Behavior |
|-------|----------|
| `Twitch.Player.ONLINE` | Hide GIF overlay, show Twitch player |
| `Twitch.Player.OFFLINE` | Hide Twitch player, show GIF overlay |
| `Twitch.Player.READY` | Player DOM ready — attach event listeners |

## Realtime Database Rules (Recommended)
Set in Firebase Console → Realtime Database → Rules:
```json
{
  "rules": {
    "messages": {
      ".read": true,
      ".write": true
    }
  }
}
```

## Data Structure (`messages/<auto-generated-key>`)
```json
{
  "text": "Hello from the audience",
  "timestamp": 1718208000000,
  "status": "pending"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `text` | string | The user-submitted text |
| `timestamp` | number | Unix milliseconds at submission time |
| `status` | string | `"pending"` = not yet read by TTS; `"read"` = TTS has spoken it |

## Browser Compatibility
- Modern browsers (Chrome 90+, Firefox 90+, Safari 15+, Edge 90+)
- ES modules required (Firebase modular SDK)
- No polyfills needed

## Security Notes
- Firebase `apiKey` is a public identifier, not a secret — normal for frontend SDKs
- Database security relies on Realtime Database Rules, not API key secrecy
- Button disabled during submission prevents accidental duplicate writes
- Input maxlength prevents oversized payloads
